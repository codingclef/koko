# Koko — Design Context

## Design Context

### Users
부모와 자녀로 구성된 가족 단위. 어르신은 없음. 주요 사용 맥락은 두 가지 — 이동 중 일정을 빠르게 확인하거나, 장을 보러 가는 길에 쇼핑 리스트를 열어보는 상황. 한 손 조작이 자연스러워야 하며, 빠르게 원하는 정보에 도달할 수 있어야 한다. 자녀도 사용하므로 직관성이 중요하다.

### Brand Personality
**따뜻함, 포근함, 정리됨, 차분함, 직관적** — 잘 정돈된 집 안처럼 조용하고 안정적이다. 화려하지 않지만 디테일이 살아 있다. "앱을 열었을 때 오늘도 잘 정리돼 있네" 하는 안도감이 목표. 채도가 높은 쨍한 색보다 muted 팔레트로 눈이 편안해야 한다.

### Aesthetic Direction
- **레퍼런스**: 캘린더는 TimeTree — 깔끔한 레이아웃, 색상 코딩된 카드, 정보 밀도가 높지만 복잡해 보이지 않음
- **기존 팔레트 유지**: `orange-400/500` 강조색 + `stone` 웜 그레이 중성색, 배경 `#fafaf9` / `#0f0e0d`
- **라운딩**: `rounded-xl` ~ `rounded-2xl` — 딱딱하지 않되 bubble스럽지 않은 중간 지점 유지
- **라이트/다크 모드 동등**: 다크 모드는 후순위가 아님. 모든 컴포넌트는 두 모드에서 동등하게 완성도 있어야 함
- **Anti-reference**: 과도하게 귀엽거나 스티커/이모지 남용. 어린이 앱처럼 보이는 것. 기업용 SaaS처럼 차갑고 무거운 것.

### Design Principles

1. **즉각적 스캔 가능성** — 사용자는 이동 중, 또는 마트 안에서 앱을 연다. 원하는 정보가 한눈에 들어와야 한다. 불필요한 depth, 과도한 텍스트, 숨겨진 액션을 피한다.

2. **웜 팝 (Warm Pop)** — stone 중성색 위의 orange 강조색이 이 앱의 정체성이다. 강조색은 아껴서, 그러나 확실하게 사용한다. 중요한 액션과 선택된 상태에서만 orange가 등장해야 한다.

3. **라이트/다크 동등 완성도** — 다크 모드는 단순한 색 반전이 아니다. 각 모드에서 컨트라스트, 계층, 분위기가 모두 의도적으로 설계되어야 한다.

4. **목적 있는 모션** — 이미 캘린더 슬라이드, 모달 슬라이드업이 구현되어 있다. 애니메이션은 공간적 맥락을 강화하거나 피드백을 줄 때만 사용한다. 장식적 모션은 추가하지 않는다.

5. **가족 모두 쓸 수 있는 직관성** — 복잡한 제스처나 숨겨진 인터랙션 없이, 보이는 그대로 작동한다. 아이콘은 레이블과 함께. 터치 타겟은 여유 있게.

### Layout Decisions

- **장바구니 탭**: 2열 그리드 (`grid-cols-2`), `max-w-lg` 제약 제거 → 전체 너비 `px-4` 사용
- **CTA 버튼**: 텍스트 버튼 대신 원형 `+` 버튼 (w-11 h-11 rounded-full) — 간결하고 터치하기 쉬움
- **카드형 레이아웃**: 장바구니 카드는 세로 방향, 아이콘+이름+미리보기 구조. 최소 높이 160px
- **아이템 미리보기**: 카드 하단에 최대 3개 표시, 초과분은 "+N개 더"
